## v2.0.6

### AI 助手动作现在会作用在启动器选中的运行时上

- AI 助手里的启动和停止 ComfyUI，会通过启动器后端操作当前选中的 ComfyUI 包。
- 插件安装和卸载也走同一套受管路径；涉及写入的动作仍然需要用户授权。
- 通过 AI 助手停止 ComfyUI 后，不会再把已经成功停止的动作误判成新的启动失败诊断。

### 运行环境管理更接近 Avalonia 1.5.3 的使用方式

- WinUI 运行环境页现在使用虚拟化依赖列表，批量卸载更稳。
- `pip`、`setuptools`、`wheel` 这类受保护运行时依赖不会被选中做破坏性卸载。
- 可以把 `.whl` 文件和 requirements 文件拖到运行环境页，直接安装到当前运行时。
- 页面补上了 Python 头文件安装/卸载、手动输入依赖名安装、ComfyUI 内核切换和 PyTorch 内核切换入口。

### 更新检查不再提示旧版本降级

- 便携版现在会用本机真实安装版本和线上最新 release 做比较。
- 当本机已经是 2.0.6 时，会忽略旧的 2.0.5 GitHub release，不会把它显示成可用更新。

### 发版验证证据

这次 release 通过了完整 Release 测试套件（`1810/1810`），生成了新的 Inno Setup 安装包，并用发布目录里的 `ModelFinder.exe` 在隔离 profile 下做了启动烟测。烟测中应用成功生成隔离配置和日志，启动后保持运行，日志里没有 warning/error/fatal 命中。
