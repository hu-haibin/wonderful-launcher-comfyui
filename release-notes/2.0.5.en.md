## v2.0.5

### Light theme is now ready for the WinUI main flow

- The WinUI shell now follows the Wonderful Launcher website palette across Home, Browser, Image, Tools, Settings, dialogs, flyouts, and bottom navigation.
- The custom title bar is back to the 32 px height used by earlier stable builds.
- Theme switching now keeps WebView, startup, Agent, task terminal, and Image Workspace surfaces in the correct light or dark state.
- Startup and task terminals keep a high-contrast log canvas so ANSI colors, warnings, and long ComfyUI output stay readable inside the light shell.

### WebView and startup behavior were checked on a real machine

- Embedded documentation stays readable after random theme and language switches.
- Chinese and English labels were checked in the docs WebView, launch surface, settings pages, Agent entry points, and task terminal entry points.
- Launch, cancel, and stop states were re-tested with a real local ComfyUI package.

### Image Workspace controls now match the theme and language

- Style and ratio flyouts now render with light-theme surfaces instead of falling back to dark or mismatched colors.
- Empty, signed-in, and auth-gated image states keep localized copy after switching languages.
- Confirmation dialogs and secondary Image Workspace actions were included in the theme pass.

### Release evidence

This release was verified with a real WinUI build using repeated light/dark and English/Chinese switches, startup/cancel passes, embedded docs, Image Workspace flyouts, settings pages, Agent entry points, and task terminal checks.
