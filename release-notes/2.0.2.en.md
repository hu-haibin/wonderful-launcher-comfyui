## v2.0.2

### WinUI Auto Update

- WinUI now runs the silent update pipeline on startup instead of relying on manual checks from Settings only.
- The desktop app can prepare update payloads in the background and surface a restart prompt when an update is ready.
- Update status handling is less likely to show confusing downgrade or stale-state prompts.

### Chinese UI Recovery

- Restored `System / zh-CN / en-US` language selection for the main WinUI launcher flow.
- Home, Settings, startup states, update prompts, and the most visible launcher surfaces now respect the selected language again.
- The release repository README now includes fresh real-app screenshots from the current WinUI path.

### Release Hardening

- Added a real Windows auto-update smoke path backed by a local fake feed and isolated profile state.
- Release closeout now verifies the published root entry (`ModelFinder.exe`) and the real Home -> startup -> embedded WebView path before shipping.
