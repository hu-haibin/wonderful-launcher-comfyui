## v2.0.7

### AI 助手现在可以真正修复启动环境

- AI 助手可以根据当前运行时的 `requirements.txt` 恢复缺失的 ComfyUI 核心依赖。
- PyTorch 修复现在走 WinUI 启动器链路，包括 GPU 运行时安装和 CUDA 校验。
- 修复完成后会回到首页继续启动同一个 ComfyUI 包，不需要用户猜测刚刚修的是哪个环境。

### WinUI 主题与国际化继续收口

- AI 助手回复、Markdown 代码块、工具卡片和状态文字会更稳定地跟随亮色/暗色主题。
- 功能页、设置页、图片工具、WebView 表面和小标签补了一轮中英文国际化。
- 图片查看器与发送到 Photoshop 的界面文案改为走统一语言服务，不再散落硬编码文本。

### 面向 .NET 10 / WinUI 的发布包更稳

- 公开 WinUI 应用现在改为 .NET framework-dependent 发布，同时继续随包携带 Windows App SDK runtime。
- WinUI 应用关闭 `PublishTrimmed` 和 `PublishReadyToRun`，避开 self-contained 构建下观察到的 .NET 10 / CsWinRT 启动崩溃路径。
- Setup 安装包会在继续安装前检查 Microsoft .NET Desktop Runtime 10.x；如果系统缺少运行时，会提示前往官方下载。

### 发版验证证据

这次 release 通过了完整 Release 测试套件（`1810/1810`），生成了 Setup 安装包，并从发布目录根入口 `ModelFinder.exe` 在隔离 profile 下做了启动烟测。烟测中外层入口成功交接到 `app/ModelFinder.App.exe`，应用启动后保持运行，没有 Windows 崩溃事件，也没有启动异常关键字命中。
