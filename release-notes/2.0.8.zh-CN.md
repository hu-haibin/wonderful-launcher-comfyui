## v2.0.8

### Setup 安装包会自动处理 .NET Desktop Runtime

- 2.0.7 安装包在缺少 Microsoft .NET Desktop Runtime 10.x 的机器上，会直接停在硬性提示弹窗。
- 2.0.8 保留更稳的 WinUI framework-dependent 发布形态，但 Setup 安装包现在内置 Microsoft .NET Desktop Runtime 10.x (x64)。
- 如果系统缺少运行时，安装器会先自动安装，再继续安装 ModelFinder。
- 如果自动安装失败，安装器会打开微软官方下载页，不再把用户卡在死路弹窗里。

### 发版流程加固

- `scripts/publish-release.ps1` 现在会在正式发版构建中下载 runtime 前置安装器，并传给 Inno Setup 打包。
- release closeout skill 已补充规则：发布时必须把 publish 脚本当作安装器契约；运行时前置条件不能破坏旧版本的用户无感体验。
- 源码守卫测试已更新，后续 WinUI framework-dependent 发版会继续校验安装器和发布脚本是否对齐。

### 包含 2.0.7 的修复

- AI 助手可以修复缺失的 ComfyUI 核心依赖和 PyTorch 运行时。
- WinUI 的 AI 回复、功能页、图片工具和 WebView 表面补齐主题与国际化修复。
- AI 助手触发依赖和 PyTorch 修复后，运行环境页会刷新状态。

### 发版验证证据

这次 release 通过了完整 Release 测试套件（`1810/1810`），通过 `scripts/publish-release.ps1` 生成了 Setup 安装包，并从发布目录根入口 `ModelFinder.exe` 做了启动烟测。烟测中外层入口成功交接到 `app/ModelFinder.App.exe`，没有 Windows 崩溃事件。
