## v2.0.6

### Agent actions now operate through the selected launcher runtime

- Assistant start and stop requests now use the launcher backend for the currently selected ComfyUI package.
- Plugin install and uninstall actions are routed through the same managed launcher path, with user approval for write operations.
- Stopping ComfyUI from the Agent no longer feeds a stale launch-failure diagnosis back into the conversation after the stop already succeeded.

### Environment management is closer to the Avalonia 1.5.3 workflow

- The WinUI environment page now uses a virtualized package list and supports safer batch uninstall flows.
- Protected runtime packages such as `pip`, `setuptools`, and `wheel` cannot be selected for destructive uninstall actions.
- You can drag `.whl` files and requirements files onto the environment page to install them into the selected runtime.
- The page now exposes Python development header install/remove actions, manual package install input, ComfyUI core switching, and PyTorch runtime switching.

### Update checks no longer offer stale downgrades

- Portable builds now compare the real installed version against the latest public release before prompting.
- A 2.0.6 build will ignore the older 2.0.5 GitHub release instead of presenting it as an available update.

### Release evidence

This release passed the full Release test suite (`1810/1810`), produced a new Inno Setup installer, and was smoke-tested from the published `ModelFinder.exe` under an isolated profile. The smoke run created its isolated config/log files, kept the published app alive after startup, and produced no warning/error/fatal log entries.
