## v2.0.7

### AI Assistant startup repair is now practical for broken ComfyUI runtimes

- The Assistant can restore missing ComfyUI core dependencies from the selected runtime's `requirements.txt`.
- PyTorch repair now runs through the WinUI launcher path, including GPU runtime installation and CUDA validation.
- After repair, the flow returns to Home so the same ComfyUI package can be started again without guessing which environment changed.

### Theme and localization fixes across WinUI

- Assistant replies, markdown blocks, tool cards, and status text now follow light/dark theme changes more consistently.
- Feature pages, settings, image tools, WebView surfaces, and small labels received another pass for untranslated English/Chinese strings.
- Image viewer and Photoshop-send UI text now uses the shared localization service instead of hardcoded strings.

### Release packaging hardened for .NET 10 / WinUI

- The public WinUI app is now published as a .NET framework-dependent payload while still bundling the Windows App SDK runtime.
- `PublishTrimmed` and `PublishReadyToRun` are disabled for the WinUI app to avoid the .NET 10 / CsWinRT startup crash path seen in self-contained builds.
- The Setup Installer now checks for Microsoft .NET Desktop Runtime 10.x before installation continues and points users to the official runtime download if it is missing.

### Release evidence

This release passed the full Release test suite (`1810/1810`), built both the Setup Installer and portable zip, and was smoke-tested from the published root `ModelFinder.exe` under an isolated profile. The smoke run handed off to `app/ModelFinder.App.exe`, kept the published app alive after startup, and produced no crash events or startup exception matches.
