## v2.0.8

### Setup Installer now handles .NET Desktop Runtime automatically

- The 2.0.7 installer could stop at a hard Microsoft .NET Desktop Runtime 10.x prompt on machines without that runtime.
- 2.0.8 keeps the safer framework-dependent WinUI package, but the Setup Installer now embeds Microsoft .NET Desktop Runtime 10.x (x64).
- If the runtime is missing, setup installs it automatically before installing ModelFinder.
- If automatic installation fails, setup opens the official Microsoft download page so the user is not left at a dead-end dialog.

### Release process hardening

- `scripts/publish-release.ps1` now downloads and passes the runtime prerequisite into Inno Setup as part of the official release build.
- The release closeout skill now explicitly treats the publish script as the installer contract and requires runtime prerequisite changes to preserve the previous user experience.
- Source guard tests were updated so future WinUI framework-dependent releases keep the installer and release script aligned.

### Included from 2.0.7

- AI Assistant startup repair for missing ComfyUI core dependencies and PyTorch runtime repair.
- WinUI theme and localization fixes across Assistant replies, feature pages, image tools, and WebView surfaces.
- Environment-page refresh after Assistant-triggered dependency and PyTorch repairs.

### Release evidence

This release passed the full Release test suite (`1810/1810`), built the Setup Installer through `scripts/publish-release.ps1`, and was smoke-tested from the published root `ModelFinder.exe`. The smoke run handed off to `app/ModelFinder.App.exe` and produced no crash events.
