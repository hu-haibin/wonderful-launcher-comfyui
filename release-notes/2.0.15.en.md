# ModelFinder 2.0.15

Released on May 21, 2026.

This release focuses on making the AI Assistant more reliable for real ComfyUI repair work, especially missing custom-node repair, environment mutation safety, and privacy-preserving feedback replay.

## What's New

- **Missing-node repair is now snapshot-bound**: the Assistant keeps the missing-node plan it just detected and installs from that same snapshot, instead of losing the plan when ComfyUI is stopped or restarted.
- **Repair task progress is easier to trust**: missing-node and plugin repair tools now report structured status such as attempted action, reason, snapshot id, terminal job id, retryability, and recommended next action.
- **Agent and manual app operations no longer fight over the same environment**: write operations use an environment operation coordinator, so plugin installs, runtime start/stop/restart, environment switching, and repair tools do not run over each other.
- **The Agent panel is less jumpy during context collection**: fast environment reads stay silent, while slower reads show a delayed inline status instead of a flashing overlay.
- **Local preferences are clearer**: the old memory control is now presented as local preferences and experience, with clearer wording about whether anything will affect the next reply.
- **Feedback now supports redacted session replay**: when allowed, Agent sessions can sync a privacy-filtered conversation replay, tool sequence, repair timeline, task type, and outcome. Solved/unsolved feedback now labels the same session instead of being the only upload trigger.

## Who Should Update

Update if you use the AI Assistant for missing custom nodes, plugin repair, ComfyUI startup diagnosis, or any workflow where the launcher and Agent may both interact with the same environment.
