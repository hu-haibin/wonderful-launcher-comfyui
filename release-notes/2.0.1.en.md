## v2.0.1

### WinUI Startup Reliability

- Fixed a WinUI startup path that could mark ComfyUI as failed while the runtime was still warming up.
- WinUI now follows the Avalonia-style startup flow more closely: spawn or attach first, then settle readiness in the background.
- Slow instances with heavy custom nodes or Manager startup work are less likely to get stuck on a false `Startup Failed` page.

### ComfyUI Desktop Path Handling

- Fixed ComfyUI Desktop launches so the launcher keeps using the Desktop data base directory from `Documents\ComfyUI`.
- Desktop instances now keep their detected data path, and launch arguments avoid duplicate base-directory or extra-model-path settings.
- This closes the gap where the official ComfyUI Desktop app could launch successfully but the WinUI launcher could not.

### Startup Logs and Home Recovery

- Restored a full `Copy All Logs` path for startup logs so users can export the complete launch log again.
- The Home startup log panel is colorized again, supports free text selection, and auto-scrolls to the latest output.
- Returning from the embedded WebView back to `Home` no longer rebuilds the startup page in a way that can freeze the UI.
