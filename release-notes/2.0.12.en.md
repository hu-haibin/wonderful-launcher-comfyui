# ModelFinder 2.0.12

Released on May 18, 2026.

This release focuses on WinUI Agent repair reliability, clearer repair progress, safer environment refresh behavior, and privacy-scoped Agent feedback data.

## What's New

- **Agent takes over startup failures more directly**: when ComfyUI fails to start, the WinUI app now hands the failure evidence to the Agent repair flow instead of leaving the user to copy logs or restart manually.
- **Clearer repair progress**: the Agent panel now explains what it is checking or changing during tool execution, so users can see the repair path instead of only a list of tool names.
- **Less risk of UI waits blocking repair**: environment refresh and post-tool context refresh paths now time out safely so a slow UI refresh does not keep the Agent tool chain stuck forever.
- **More practical dependency repair**: the WinUI environment page keeps `pip` protected but no longer blocks PyTorch-related packages from repair workflows that intentionally need to replace them.
- **Sanitized Agent feedback context**: when users voluntarily submit Agent feedback, the desktop app can send a capped, privacy-sanitized conversation digest to improve future diagnostics.
- **Avalonia classic interface restored**: the Avalonia desktop shell has been restored to the older 1.5.3-style interface for users who rely on that build path.

## Who Should Update

Update if you use the WinUI desktop build with Agent-assisted startup repair, dependency repair, missing-node triage, or feedback reporting.
