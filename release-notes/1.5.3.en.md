# ModelFinder 1.5.3

This update mainly makes the most common repair flows more reliable: fixing environments, dependencies, and plugin installs.

## What's new

- The AI Assistant can now take over more real-world failures  
  In addition to ComfyUI startup failures, it can now jump in when a single Python package install fails or when a single plugin install fails.

- Multi-step repair feels smoother after one approval  
  When the AI Assistant needs to run several repair steps in the same repair flow, the approval experience is clearer and less interruptive.

- Smarter plugin installation  
  Plugin install now checks local plugin data first, then falls back to online repository lookup when needed. Plugins like `InstantID` are easier to resolve and continue installing.

- Better sign-in, recharge, and credit visibility in AI chat  
  You can now see your credits directly in the AI chat header, and the balance refreshes automatically after credits are charged or refunded. The Settings page and AI chat now stay in sync.

- Fixed 7-Zip download failures  
  Replaced the broken 7-Zip mirror path. If a download still fails, the app now tells you which download sources were attempted, so it is easier to tell whether the problem is the mirror or the network.

## What this means for you

- Startup failures, dependency issues, and plugin install failures are easier to recover from
- The AI Assistant is more likely to find the right plugin repository on its own
- Credit balance is more trustworthy and updates more consistently
- Basic dependency downloads like 7-Zip should succeed more often and fail more clearly
